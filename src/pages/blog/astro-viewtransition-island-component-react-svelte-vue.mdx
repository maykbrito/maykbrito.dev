## Entendendo o problema


Uma das estratégias mais legais da ViewTransition API do [Astro](https://astro.build/), é a sacada da gente transicionar de um elemento em um página, para outro de mesmo ID em outra página, fazendo com que haja uma continuidade de transição, é muito bacana!


Apesar de ser comum usarmos a propriedade `transition:name` em alguma tag dentro de arquivos `.astro`, essa estratégia não funciona se usarmos dentro de um componente como React, Svelte ou Vue. Veja


```javascript
/* src/pages/index.astro */
<a href="/about" transition:name={`cover-title-${frontmatter.id}`}>Sobre mim</a>

/* src/pages/about.astro */
<h1 transition:name={`cover-title-${frontmatter.id}`}>Título</h1>
```


## Solução


Para resolver é simples. Iremos utilizar a propriedade `view-transition-name` em um estilo dinâmico, veja:


```javascript
/* Imagina que o código abaixo esteja dentro de um component React, por exemplo */
<div style={`view-transition-name: cover-title-${frontmatter.id}`}>Título</div>

/* src/pages/about.astro */
<h1 style={`view-transition-name: cover-title-${id}`}>Título</h1>
```
