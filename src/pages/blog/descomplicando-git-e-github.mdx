## O que é um controle de versão?


Antes da gente falar do Git, vamos dar um passo atrás e entender o motivo dele existir e eu gosto de fazer a analogia do controle do tempo!

De maneira semelhante, um sistema de controle de versão permite aos desenvolvedores "manipular" o tempo e as mudanças em seus projetos, mas como?

Assim como Dr. Estranho pode ver diferentes futuros e escolher o caminho mais adequado, um sistema de controle de versão permite que os desenvolvedores rastreiem as alterações feitas em um projeto ao longo do tempo. Eles podem voltar a versões anteriores, criar "ramificações" para experimentar com novas funcionalidades sem afetar a versão principal (semelhante a diferentes realidades), e depois decidir se desejam mesclar essas mudanças de volta ao projeto principal.


Portanto, assim como Dr. Estranho usa seus poderes para moldar o curso dos eventos, um sistema de controle de versão dá aos desenvolvedores a capacidade de moldar e gerenciar o histórico de seus projetos, proporcionando flexibilidade e controle sobre o desenvolvimento.


## O que é o Git?


O Git é um sistema de controle de versão que permite acompanhar as alterações feitas nos seus arquivos ao longo do tempo. Com o Git, é possível voltar a vários estados dos seus arquivos, como se fosse uma máquina do tempo. Além disso, você pode criar uma cópia do seu arquivo, fazer alterações nessa cópia e depois mesclar essas mudanças ao arquivo original.


Por exemplo, imagine que você esteja trabalhando na página inicial de um site e perceba que não gosta da barra de navegação. No entanto, ao mesmo tempo, talvez você não queira começar a alterar seus componentes, pois isso poderia piorar a situação.


Com o Git, você pode criar uma cópia idêntica desse arquivo e experimentar com a barra de navegação. Então, quando estiver satisfeito com suas alterações, pode mesclar a cópia com o arquivo original.


Você não está limitado a usar o Git apenas para arquivos de código-fonte - também pode usá-lo para acompanhar arquivos de texto ou até mesmo imagens. Isso significa que o Git não é exclusivo para desenvolvedores - qualquer pessoa pode achar útil.
